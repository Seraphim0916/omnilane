# Live UI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use
> `superpowers:executing-plans` and `superpowers:test-driven-development`.
> Complete tasks in order and preserve the read-only boundary.

**Goal:** Add an optional local-only web board that observes current and past
omnilane jobs without adding browser-side dispatch or a second history store.

**Architecture:** A Python 3.9+ standard-library server reads the existing
`$OMNILANE_HOME/jobs` contract. A protected lifecycle state file and
cross-process lock own one loopback listener. Static local assets fetch
authenticated JSON detail and subscribe to cached SSE summaries. The browser
never mutates jobs or renders model text as HTML.

**Tech stack:** Python 3.9 stdlib, `ThreadingHTTPServer`, SSE, HTML, CSS,
vanilla JavaScript, Bash entrypoint, `unittest`.

## Global constraints

- Bind only `127.0.0.1`; enforce the exact Host header.
- Require a 256-bit random token on every `/api/` request.
- Never serve raw logs, environment data, hidden reasoning, or task/output
  bodies in list/SSE summaries.
- Expose no HTTP mutation route and launch no process from request handlers.
- Refuse symlinked runtime/job paths and cap all reads before parsing.
- Serialize `start`/`stop` with a cross-process lifecycle lock.
- Reuse the existing jobs directory; do not add SQLite or another event store.
- Keep all frontend resources local and render untrusted text with text nodes.
- Every behavior change starts with a failing focused test.
